# Day 04 - Create Azure Virtual Network (VNet)

## Objective
Create a Virtual Network named `xfusion-vnet` in the West US region.

## Steps Performed
1. Logged in to the Azure Portal.
2. Navigated to **Virtual Networks**.
3. Created a new Virtual Network named `xfusion-vnet`.
4. Selected the **West US** region.
5. Configured an IPv4 address space (`10.0.0.0/16`).
6. Reviewed the configuration and deployed the VNet.

## Outcome
Successfully created the `xfusion-vnet` Virtual Network in the West US region.

## Key Learnings
- Azure Virtual Networks provide private networking for Azure resources.
- A VNet requires an IPv4 address space defined using CIDR notation.
- VNets can be divided into subnets for better network organization and security.
