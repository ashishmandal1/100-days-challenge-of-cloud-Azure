# Day 05 – Azure Virtual Network (VNet)

## Objective

Create an Azure Virtual Network (VNet) to establish a private network for Azure resources.

## Services Used

* Azure Virtual Network (VNet)

## Task Performed

* Logged in to the Azure Portal using the lab credentials.
* Opened the **Virtual Networks** service.
* Created a Virtual Network named **datacenter-vnet**.
* Selected the **East US** region.
* Configured the IPv4 address space as **192.168.0.0/24**.
* Added a subnet to the virtual network.
* Reviewed the configuration and successfully deployed the Virtual Network.

## What I Learned

* Azure Virtual Networks provide secure and isolated networking for Azure resources.
* A VNet requires an address space defined using CIDR notation.
* Subnets divide a VNet into smaller network segments for better organization and resource management.
* Choosing an appropriate address space is essential for future scalability and connectivity.
